## Key Implementation Notes

- **No backend changes needed**: `DATABASE_URL` env var already supported in `config.py`
- **seed.py only creates users**: Tests create all test data via API helpers
- **No data-testid exist yet**: Use `getByRole`, `getByLabel`, `getByText`, CSS selectors initially; add `data-testid` incrementally when needed
- **JWT in sessionStorage**: `context.addInitScript` to inject token
- **CSV via blob**: Use `page.waitForEvent('download')` pattern
- **Status transitions require optimistic locking**: Fetch current `updated_at` before each transition
- **Status machine maps** (from StatusTransitionButton.jsx):
  - Permission: draft→申請, submitted→取り下げ/審査開始, under_review→承認/差戻し, pending_approval→承認/差戻し, approved→交付, rejected→再申請
  - Lease: draft→協議開始, negotiating→承認申請, pending_approval→契約開始/差戻し, active→解約
